Agent Teams 文 档 分 析 报 告 
基 本 信 息 
- 标 题 : Orchestrate teams of Claude Code sessions
- URL: https://code.claude.com/docs/en/agent-teams
- 分 析 时 间 : 2026年 2月 6日  22:51
核 心 摘 要 
本 文 档 介 绍 了  Claude Code 的  Agent Teams 功 能 ，该 功 能 允 许 协 调 多 个  Claude Code 实 例 作 为 一 个 团 队 协 作 工 作 。团 队 由 一 个  team lead 和 多 个  teammates 组 成 ，支 持 共 享 任 务 列 表 、跨  agent 消 息 传 递 和 集 中 化 管 理 。Agent Teams 适 用 于 需 要 并 行 探 索 的 复 杂 任 务 ，但 这 是 一 个 实 验 性 功 能 ，默 认 禁 用 ，需 要 通 过 环 境 变 量 启 用 。
内 容 结 构 
主 要 章 节 
1. 何 时 使 用  Agent Teams - 最 佳 用 例 和 与  subagents 的 对 比 
2. 启 用  Agent Teams - 配 置 方 法 
3. 启 动 第 一 个  Agent Team - 创 建 和 使 用 流 程 
4. 控 制  Agent Team - 显 示 模 式 、任 务 分 配 、委 托 等 
5. Agent Teams 工 作 原 理  - 架 构 和 机 制 
6. 用 例 示 例  - 实 际 应 用 场 景 
7. 最 佳 实 践  - 使 用 建 议 
8. 故 障 排 除  - 常 见 问 题 解 决 
9. 局 限 性  - 当 前 限 制 
核 心 论 点 
- Agent Teams 擅 长 处 理 需 要 并 行 探 索 的 任 务 
- 与  subagents 的 区 别 在 于 团 队 成 员 可 以 直 接 相 互 通 信 
- 增 加 了 协 调 开 销 和  token 成 本 
- 适 用 于 独 立 并 行 操 作 的 场 景 ，不 适 合 顺 序 任 务 或 依 赖 性 强 的 任 务 
关 键 知 识 点 
1. Agent Teams vs Subagents
| 特 性  | Subagents | Agent Teams |
|------|-----------|-------------|
| 上 下 文  | 独 立 ，结 果 返 回 给 调 用 者  | 独 立 ，完 全 独 立 运 行  |
| 通 信  | 只 向 主  agent 报 告  | 团 队 成 员 直 接 相 互 通 信  |
| 协 调  | 主  agent 管 理 所 有 工 作  | 共 享 任 务 列 表 ，自 我 协 调  |
| 最 佳 用 途  | 需 要 结 果 的 专 注 任 务  | 需 要 讨 论 和 协 作 的 复 杂 工 作  |
| Token 成 本  | 较 低  | 较 高  |
2. 最 佳 使 用 场 景 
- 研 究 和 审 查 : 多 个  teammates 同 时 调 查 问 题 的 不 同 方 面 
- 新 模 块 或 功 能 : 每 个  teammate 拥 有 独 立 的 部 分 
- 竞 争 假 设 调 试 : teammates 并 行 测 试 不 同 理 论 
- 跨 层 协 调 : 前 端 、后 端 和 测 试 的 更 改 分 别 由 不 同  teammate 拥 有 
3. 架 构 组 件 
- Team lead: 创 建 团 队 、生 成  teammates、协 调 工 作 的 主 会 话 
- Teammates: 在 分 配 任 务 上 工 作 的 独 立  Claude Code 实 例 
- Task list: teammates 声 明 和 完 成 的 工 作 项 共 享 列 表 
- Mailbox: agent 之 间 通 信 的 消 息 系 统 
4. 显 示 模 式 
- In-process: 所 有  teammates 在 主 终 端 内 运 行 （默 认 ）
- Split panes: 每 个  teammate 有 独 立 窗 格 （需 要  tmux 或  iTerm2）
5. 核 心 功 能 
- 任 务 依 赖 管 理 : 系 统 自 动 管 理 任 务 依 赖 关 系 
- 文 件 锁 定 防 止 竞 争 : 多 个  teammates 不 会 同 时 声 明 同 一 任 务 
- 直 接 消 息 传 递 : 使 用  Shift+Up/Down 选 择  teammate 并 直 接 消 息 
- 委 托 模 式 : 限 制  lead 只 使 用 协 调 工 具 
重 要 概 念 
Team Lead
主 要 的  Claude Code 会 话 ，负 责 创 建 团 队 、生 成  teammates 和 协 调 工 作 。
Teammate
独 立 的  Claude Code 实 例 ，每 个 在 各 自 的 上 下 文 窗 口 中 工 作 。
Shared Task List
团 队 成 员 可 以 查 看 和 声 明 的 任 务 共 享 列 表 ，支 持 任 务 依 赖 关 系 。
Mailbox System
agent 之 间 通 信 的 消 息 传 递 系 统 ，支 持 点 对 点 消 息 和 广 播 。
Delegate Mode
限 制  lead 只 能 使 用 协 调 工 具 （生 成 、消 息 传 递 、关 闭  teammates、管 理 任 务 ）的 模 式 。
数 据 与 事 实 
配 置 要 求 
- 环 境 变 量 : CLAUDECODEEXPERIMENTALAGENTTEAMS=1
- 设 置 位 置 : settings.json 或  shell 环 境 
存 储 位 置 
- Team config: /.claude/teams/{team-name}/config.json
- Task list: /.claude/tasks/{team-name}/
Token 使 用 
- 每 个  teammate 有 独 立 的 上 下 文 窗 口 
- Token 使 用 量 与 活 跃  teammates 数 量 成 正 比 
- 研 究 、审 查 和 新 功 能 工 作 通 常 值 得 额 外 成 本 
当 前 限 制 
1. In-process teammates 无 法 恢 复 会 话 （/resume 和  /rewind 不 工 作 ）
2. 任 务 状 态 可 能 滞 后 
3. 关 闭 可 能 很 慢 
4. 每 个 会 话 只 能 有 一 个 团 队 
5. 不 支 持 嵌 套 团 队 
6. teammates 不 能 生 成 自 己 的 团 队 
7. permissions 在 生 成 时 设 置 
8. 分 屏 模 式 需 要  tmux 或  iTerm2
延 伸 阅 读 
- [Subagents 文 档 ](/en/sub-agents) - 轻 量 级 委 托 方 法 
- [Settings 文 档 ](/en/settings) - 配 置 指 南 
- [Permissions 文 档 ](/en/permissions) - 权 限 设 置 
- [Agent team token costs](/en/costs#agent-team-token-costs) - 成 本 指 导 
- [Git worktrees](/en/common-workflows#run-parallel-claude-code-sessions-with-git-worktrees) - 手 动 并 行 会 话 
实 用 示 例 
创 建 并 行 代 码 审 查 团 队 
Create an agent team to review PR #142. Spawn three reviewers:
- One focused on security implications
- One checking performance impact
- One validating test coverage
Have them each review and report findings.
竞 争 假 设 调 查 
Users report the app exits after one message instead of staying connected.
Spawn 5 agent teammates to investigate different hypotheses. Have them talk to
each other to try to disprove each other's theories, like a scientific
debate. Update the findings doc with whatever consensus emerges.
创 建 设 计 团 队 
I'm designing a CLI tool that helps developers track TODO comments across
their codebase. Create an agent team to explore this from different angles: one
teammate on UX, one on technical architecture, one playing devil's advocate.
最 佳 实 践 总 结 
给  teammates 足 够 的 上 下 文 
- Teammates 会 自 动 加 载 项 目 上 下 文 （CLAUDE.md、MCP 服 务 器 、skills）
- 但 不 会 继 承  lead 的 对 话 历 史 
- 在 生 成 提 示 中 包 含 任 务 特 定 细 节 
合 理 设 置 任 务 大 小 
- 太 小 : 协 调 开 销 超 过 收 益 
- 太 大 : teammates 工 作 时 间 太 长 没 有 检 查 点 
- 合 适 : 自 包 含 单 元 ，产 生 清 晰 的 交 付 成 果 
等 待  teammates 完 成 
如 果  lead 开 始 自 己 实 现 任 务 而 不 是 等 待  teammates：
Wait for your teammates to complete their tasks before proceeding
从 研 究 和 审 查 开 始 
新 用 户 建 议 从 有 清 晰 边 界 的 任 务 开 始 ：
- 审 查  PR
- 研 究 库 
- 调 查  bug
避 免 文 件 冲 突 
确 保 每 个  teammate 拥 有 不 同 的 文 件 集 
监 控 和 引 导 
- 定 期 检 查  teammates 进 度 
- 重 定 向 不 工 作 的 方 法 
- 及 时 综 合 发 现

Claude Code 2026 超 详 细 安 装 教 程 ｜Windows/Mac 一 站 式 搞 定 ，新 手 必 看 ！
> 导 读 ：2026 年 了 ，Claude Code 已 经 成 为 最 强  AI 编 程 助 手 之 一 。很 多 小 伙 伴 问 怎 么 安 装 ，今 天 这 篇 教 程 从 零 基 础 开 始 ，手 把 手 教 你 安 装 配 置 ，保 证 看 完 就 会 ！
---
📋 Before You Start｜安 装 前 准 备 
在 开 始 之 前 ，先 确 认 几 件 事 ：
✅ 必 要 条 件 
| 项 目  | 要 求  | 检 查 方 法  |
|------|------|----------|
| 操 作 系 统  | Windows 10+/macOS 10.15+/Linux | - |
| 网 络 环 境  | 能 访 问  claude.ai | 浏 览 器 打 开 试 试  |
| API Key | 需 要  Anthropic 账 号  | 没 账 号 先 去 注 册  |
| 磁 盘 空 间  | 约  500MB | 应 该 都 够  |
⚠️ Windows 用 户 特 别 注 意 
必 须 先 安 装  Git！ 这 是  2026 年 官 方 安 装 脚 本 的 依 赖 。
下 载 安 装 地 址 ：https://git-scm.com/download/win
安 装 时 一 路 默 认 即 可 ，记 住 安 装 路 径 （通 常 是  C:\Program Files\Git）。
---
🚀 安 装 方 式 一 ：官 方 推 荐 （原 生 安 装 ）⭐
这 是  2026 年 官 方 最 推 荐 的 方 式 ，无 需 安 装  Node.js，更 稳 定 更 快 速 ！
1️⃣ Windows PowerShell 用 户 （推 荐 ）
一 行 命 令 搞 定 ：
powershell
irm https://claude.ai/install.ps1 | iex
操 作 步 骤 ：
1. 右 键 点 击 开 始 菜 单  → 选 择  Windows PowerShell 或  终 端 
2. 复 制 上 面 命 令 ，粘 贴 进 去 
3. 按 回 车 ，等 待 下 载 安 装 
4. 安 装 完 成 后 重 启 终 端 
2️⃣ Windows CMD/命 令 提 示 符 用 户 
cmd
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
3️⃣ macOS / Linux 用 户 
bash
curl -fsSL https://claude.ai/install.sh | bash
4️⃣ 验 证 安 装 成 功 
打 开 新 的 终 端 窗 口 ，输 入 ：
bash
claude --version
如 果 显 示 版 本 号 （如  2.1.x 或 更 高 ），恭 喜 ！安 装 成 功 ！🎉
---
📦 安 装 方 式 二 ：npm 安 装 （备 选 方 案 ）
> ⚠️ 注 意 ：npm 方 式 已 被 官 方 标 记 为  Deprecated（弃 用 ），但 依 然 可 用 。如 果 你 已 经 装 了  Node.js 或 者 在 某 些 特 殊 环 境 只 能 用  npm，可 以 参 考 这 种 方 式 。
前 置 条 件 
确 保 已 安 装  Node.js 18+（推 荐  20+ 或  22 LTS）
检 查 版 本 ：
bash
node -v
没 有 的 话 去  https://nodejs.org 下 载 安 装 。
安 装 命 令 
bash
npm install -g @anthropic-ai/claude-code
国 内 用 户 如 果 遇 到 网 络 问 题 ，使 用 国 内 镜 像 ：
bash
npm install -g @anthropic-ai/claude-code --registry https://registry.npmmirror.com
验 证 
bash
claude --version
---
🔄 从  npm 迁 移 到 原 生 版 本 （老 用 户 必 看 ）
如 果 你 之 前 用  npm 安 装 的 ，官 方 建 议 迁 移 到 原 生 版 本 。迁 移 过 程 很 简 单 ：
第  1 步 ：备 份 配 置 （强 烈 建 议 ！）
bash
macOS / Linux
cp -r /.claude /.claude-backup-$(date +%Y%m%d)
Windows PowerShell
Copy-Item -Recurse /.claude /.claude-backup-$(Get-Date -Format yyyyMMdd)
第  2 步 ：运 行 迁 移 命 令 
bash
claude install
这 个 命 令 会 自 动 ：
- 检 查 并 安 装 原 生 版 本 
- 迁 移 你 的 配 置 、认 证 信 息 、历 史 记 录 
- 把  native binary 放 到 合 适 的 位 置 （通 常 是  /.local/bin）
第  3 步 ：验 证 迁 移 
关 闭 终 端 重 新 打 开 ，然 后 ：
bash
claude --version          应 该 看 到  2.x.x 新 版 本 
which claude              查 看 路 径 （通 常 在  /.local/bin）
claude doctor             运 行 诊 断 检 查 
第  4 步 ：卸 载 旧 版 本 （可 选 但 推 荐 ）
确 认 迁 移 成 功 后 ，卸 载  npm 版 本 ：
bash
npm uninstall -g @anthropic-ai/claude-code

💡 常 见 问 题  FAQ

试 试 这 几 招 ：

3. 关 闭 杀 毒 软 件 /防 火 墙 临 时 试 试

Claude Code 与  App 开 发 对 话 记 录

---
一 、开 发 能 够 上 线 使 用 的  App 需 要 哪 些 操 作 流 程 ？
1. 需 求 阶 段 
- 明 确 产 品 定 位 与 核 心 功 能 
- 市 场 /用 户 调 研 
- 撰 写 需 求 文 档  / PRD
2. 设 计 阶 段 
- 架 构 设 计 （前 端 /后 端 /数 据 库 /部 署 方 案 ）
- UI/UX 设 计 
- API 设 计 
3. 开 发 阶 段 
- 编 码 实 现 
- 代 码 审 查 （Code Review）
- 单 元 测 试  / 集 成 测 试 
4. 测 试 阶 段 
- 功 能 测 试 
- 性 能 /压 力 测 试 
- 安 全 测 试 
- UAT（用 户 验 收 测 试 ）
5. 部 署 准 备 
- 配 置 生 产 环 境 
- 数 据 迁 移 方 案 
- 监 控 /日 志 告 警 接 入 
- 备 份 /回 滚 策 略 
6. 上 线 发 布 
- 灰 度 发 布  / 分 批 发 布 
- 快 速 迭 代 修 复 问 题 
- 正 式 全 量 发 布 
7. 运 维 监 控 
- 线 上 监 控 （Django/Nginx/数 据 库 等 ）
- 故 障 响 应 机 制 
- 定 期 更 新 迭 代 
---

四 、有 没 有 开 发 特 定  App 的  Skills？
结 论 ：没 有 专 门 针 对 某 款 特 定  App 的  Skills（如 "抖 音  App 开 发  Skill"），但 有 大 量 使 用  Claude Code 开 发 各 类  App 的 实 战 案 例 。

| App 类 型  | 开 发 者  | 技 术 栈  | 成 果  |
|---------|--------|--------|------|

| 自 动 饮 品 售 卖 小 程 序  | 独 立 开 发 者  | 微 信 小 程 序  + 后 端  | 扫 码 点 单 、库 存 管 理 、支 付 集 成  |

常 用  Skills 组 合 （5个 最 实 用 的 ）
brainstorming      → 产 品 头 脑 风 暴  + 选 择 题 式 需 求 确 认 
frontend-design    → UI 美 化 （减 少  AI 味 ）
superpowers        → 强 制  TDD 流 程 
impeccable         → 界 面 细 节 优 化 （布 局 /间 距 /视 觉 层 级 ）
核 心 发 现 
Claude Code 不 是 "针 对 特 定  App 的  Skill"，而 是 一 个 通 用 编 程  Agent。 它 的 强 大 之 处 在 于 ：
1. 自 然 语 言  → 全 栈 代 码 （前 端 /后 端 /数 据 库 ）
2. 持 续 对 话 式 迭 代 （提 需 求  → 看 效 果  → 反 馈  → 改 代 码 ）
3. 配 合  Skills 使 用 （UI设 计 、代 码 审 查 、测 试 等 专 项 增 强 ）
典 型 工 作 流

  ↓

  ↓
UI 设 计 （frontend-design / impeccable）
  ↓
代 码 开 发 （Claude Code 直 接 生 成 ）
  ↓
测 试  & 修 复 循 环

五 、总 结 
Claude Code 可 以 在  App 开 发 的 编 码 实 现 、代 码 审 查 、单 元 测 试 、问 题 排 查 、架 构 设 计 等 环 节 提 供 强 有 力 协 助 。
但 上 线 部 署 、应 用 市 场 上 架 、云 服 务 管 理 等 环 节 仍 需 要 你 自 行 完 成 。
关 键 不 是 找 特 定 的  App 开 发  Skill，而 是 明 确 你 想 做 什 么  App，Claude Code 配 合 合 适 的  Skills 可 以 帮 你 从 零 实 现 。

- 写 新 功 能 ：帮 我 在  src/api/ 下 加 一 个 获 取 用 户 详 细 信 息 的 接 口 ，要 带 鉴 权 和 错 误 处 理

- 写 测 试 ：给  utils/date.js 里 的 所 有 函 数 都 补 上 单 元 测 试 
如 何 安 全 迁 移 （推 荐 步 骤 ，2026 年 最 新 流 程 ）
1. 先 备 份 配 置 （强 烈 建 议 ）  
   配 置 基 本 都 在  C:\Users\用 户 名 \.claude 目 录 下 ，迁 移 会 尽 量 保 留 ，但 先 备 份 无 害 ：
   bash
   cp -r /.claude /.claude-backup-$(date +%Y%m%d)
   
2. 运 行 迁 移 命 令 （最 简 单 官 方 方 式 ）  
   直 接 在 终 端 运 行 ：
   bash
   claude install
   
   - 这 条 命 令 是 官 方 专 门 为  npm 用 户 准 备 的 迁 移 入 口 
   - 它 会 ：
     - 下 载 并 安 装  native 版 本 
     - 尝 试 保 留 你 的 设 置 、认 证 信 息 、历 史 等 
     - 把  native binary 放 到 合 适 的 位 置 （通 常  /.local/bin/claude 或 系 统  PATH 已 包 含 的 位 置 ）
   - 过 程 中 如 果 提 示 确 认 ，输 入  y 继 续 
3. 验 证 新 版 本 是 否 正 常 工 作   
   迁 移 完 成 后 ，关 闭 并 重 新 打 开 终 端 （或  source 你 的  shell 配 置 文 件 ），然 后 检 查 ：
   bash
   claude --version          应 该 看 到 类 似  2.x.x 的 新 版 本 号 
   which claude              macOS/Linux，看 看 路 径 是 否 变 了 （通 常 在  /.local/bin）
   claude doctor             运 行 健 康 检 查 ，查 看 是 否 有 问 题 
   
4. 清 理 旧 的  npm 版 本 （可 选 但 推 荐 ）  
   迁 移 成 功  + 确 认  native 版 一 切 正 常 后 ，卸 载 旧 版 避 免 冲 突 ：
   bash
   npm uninstall -g @anthropic-ai/claude-code
   
   - 如 果 之 前 用  npx 或 其 他 方 式 装 过 ，也 可 检 查  npm list -g | grep claude 清 理 残 留 
5. 如 果 迁 移 中 遇 到 问 题   
   - 卡 住 /网 络 问 题  → 直 接 用 一 键 安 装 脚 本 重 装 （会 覆 盖 ）：
     - macOS/Linux/WSL：curl -fsSL https://claude.ai/install.sh | bash
     - Windows PowerShell：irm https://claude.ai/install.ps1 | iex
   - 仍 然 提 示 旧 版 警 告  → 说 明  native 没 覆 盖 成 功 ，手 动  rm npm 版 后 重 跑  install
   - PATH 没 生 效  → 手 动 加  export PATH="$HOME/.local/bin:$PATH" 到  /.zshrc 或  /.bashrc，然 后  source
迁 移 后 你 会 发 现 启 动 快 了 很 多 ，更 新 自 动 ，报 错 少 很 多 ——这 是 目 前  Claude Code 团 队 最 希 望 用 户 走 的 路 。

核 心 摘 要 
本 文 档 详 细 介 绍  Claude Code 中 的  子 代 理  (Subagents) 功 能 ——一 种 可 以 处 理 特 定 类 型 任 务 的 专 业 化  AI 助 手 。子 代 理 拥 有 独 立 的 上 下 文 窗 口 、自 定 义 系 统 提 示 、特 定 工 具 访 问 权 限 和 独 立 权 限 ，通 过 任 务 委 派 实 现 高 效 的 代 码 库 管 理 和 开 发 工 作 流 。
---
内 容 结 构 
主 要 章 节 
1. 子 代 理 概 述  - 什 么 是 子 代 理 及 其 核 心 价 值 
2. 内 置 子 代 理  - Explore、Plan、General-purpose 等 
3. 快 速 开 始  - 创 建 第 一 个 子 代 理 
4. 配 置 选 项  - 作 用 域 、模 型 、工 具 、权 限 等 
5. 工 作 模 式  - 前 台 /后 台 、自 动 委 派 、并 行 研 究 
6. 示 例 子 代 理  - Code Reviewer、Debugger、Data Scientist 等 
核 心 论 点 
- 子 代 理 通 过 隔 离 上 下 文 保 护 主 对 话 的 整 洁 
- 通 过 工 具 限 制 实 现 安 全 约 束 
- 通 过 持 久 化 记 忆 积 累 跨 会 话 知 识 
---
关 键 知 识 点 
知 识 点  1: 子 代 理 的 核 心 价 值  ⭐⭐⭐
| 价 值 点  | 说 明  |
|--------|------|
| 保 留 上 下 文  | 将 探 索 和 实 现 工 作 隔 离 到 独 立 对 话 ，避 免 主 对 话 被 大 量 输 出 污 染  |
| 强 制 约 束  | 限 制 子 代 理 可 使 用 的 工 具 ，确 保 安 全  |
| 复 用 配 置  | 用 户 级 子 代 理 可 在 所 有 项 目 中 使 用  |
| 专 业 化 行 为  | 针 对 特 定 领 域 的 专 注 系 统 提 示  |
| 控 制 成 本  | 将 任 务 路 由 到 更 快 的 模 型 （如  Haiku） |
知 识 点  2: 四 种 作 用 域 优 先 级  ⭐⭐⭐

=== ID: 2204 | Title: GSD (Get Shit Done) - Claude Code 完整使用教程 | Path: 人工智能/claude code/GSD-Claude-Code 使用教程.md | Words: 7296 ===
GSD (Get Shit Done) - Claude Code 完 整 使 用 教 程 
> GSD 是 一 个 轻 量 级 但 强 大 的 元 提 示  (meta-prompting)、上 下 文 工 程 和 规 范 驱 动 开 发 系 统 ，专 为  Claude Code、OpenCode、Gemini CLI 和  Codex 设 计 。
>
> 核 心 作 用 : 解 决  Context Rot（上 下 文 腐 烂 ） —— 当  AI 上 下 文 窗 口 被 填 满 时 ，代 码 质 量 会 下 降 的 问 题 。
---
📦 一 、快 速 安 装 
1. 安 装  GSD
打 开 终 端 ，运 行 ：
bash
npx get-shit-done-cc@latest
安 装 程 序 会 提 示 你 选 择 ：
- 运 行 时 ：选 择  Claude Code（或  OpenCode、Gemini、Codex）
- 位 置 ：Global（所 有 项 目 ）或  Local（当 前 项 目 ）
2. 验 证 安 装 
bash
/gsd:help          Claude Code / Gemini
/gsd-help          OpenCode
$gsd-help          Codex
能 看 到 帮 助 信 息 就 说 明 安 装 成 功 了 ！
3. 推 荐 配 置 
GSD 需 要 频 繁 执 行  Git 命 令 ，建 议 使 用 无 权 限 模 式 运 行  Claude Code：
bash
claude --dangerously-skip-permissions
> 💡 为 什 么 ？ 如 果 每 次  git commit、date 都 要 确 认 ，会 非 常 打 断 工 作 流 。
---
🎯 二 、核 心 工 作 流 程 
完 整 流 程 图 
/gsd:new-project 
    ↓
/gsd:discuss-phase N 
    ↓
/gsd:plan-phase N 
    ↓
/gsd:execute-phase N 
    ↓
/gsd:verify-work N 
    ↓
/gsd:complete-milestone
---
📝 三 、详 细 使 用 步 骤 
步 骤  1️⃣：初 始 化 项 目 
bash
/gsd:new-project
系 统 会 执 行 ：
1. 提 问 阶 段  - 系 统 会 一 直 问 问 题 直 到 完 全 理 解 你 的 想 法 
   - 项 目 目 标 、功 能 需 求 
   - 技 术 栈 偏 好 
   - 约 束 条 件 和 边 界 情 况 
   
2. 市 场 /技 术 研 究  - 并 行 启 动 多 个  Agent 研 究 相 关 领 域 （可 选 但 推 荐 ）
3. 需 求 提 取  - 自 动 区 分  v1/v2 功 能 和 范 围 外 内 容 
4. 创 建 路 线 图  - 生 成 分 阶 段 的 开 发 计 划 
生 成 的 文 件 ：
- PROJECT.md - 项 目 愿 景 （始 终 加 载 ）
- REQUIREMENTS.md - 需 求 文 档 
- ROADMAP.md - 开 发 路 线 图 
- STATE.md - 决 策 和 状 态 记 录 
- .planning/research/ - 研 究 报 告 
---
步 骤  2️⃣：讨 论 阶 段 （可 选 但 推 荐 ）
bash
/gsd:discuss-phase 1
作 用 ： 在 规 划 之 前 ，捕 获 你 对 实 现 方 式 的 偏 好 
系 统 会 分 析 当 前 阶 段 ，针 对 模 糊 地 带 提 问 ：
- 视 觉 功 能  → 布 局 、密 度 、交 互 方 式 、空 状 态 
- API/CLI → 响 应 格 式 、参 数 、错 误 处 理 
- 内 容 系 统  → 结 构 、语 气 、深 度 
- 组 织 任 务  → 分 组 标 准 、命 名 规 则 
输 出 ： {phasenum}-CONTEXT.md - 这 份 文 档 会 指 导 后 续 的 研 究 和 规 划 
> 💡 提 示 ： 这 里 讨 论 得 越 详 细 ，最 终 结 果 越 符 合 你 的 预 期 。跳 过 这 步 会 得 到 "合 理 默 认 "，深 入 讨 论 会 得 到 "你 的 愿 景 "。
---
步 骤  3️⃣：规 划 阶 段 
bash
/gsd:plan-phase 1
系 统 会 执 行 ：
1. 研 究  - 调 查 如 何 实 现 这 个 阶 段 ，参 考 你 的  CONTEXT.md 决 策 
2. 规 划  - 创 建  2-3 个 原 子 任 务 计 划 （XML 结 构 ）
3. 验 证  - 检 查 计 划 是 否 符 合 需 求 ，循 环 验 证 直 到 通 过 
每 个 计 划 的 结 构 示 例 ：
xml
<task type="auto">
  <name>Create login endpoint</name>
  <files>src/app/api/auth/login/route.ts</files>
  <action>Use jose for JWT tokens...</action>
  <verify>curl -X POST http://localhost:3000/api/auth/login...</verify>
  <done>Valid credentials return cookie, invalid returns 401</done>
</task>
生 成 的 文 件 ：
- {phasenum}-RESEARCH.md - 技 术 研 究 报 告 
- {phasenum}-{N}-PLAN.md - 具 体 任 务 计 划 
---
步 骤  4️⃣：执 行 阶 段 
bash
/gsd:execute-phase 1
系 统 会 ：
1. 分 析 任 务 依 赖 关 系 
2. 并 行 波 次 执 行  - 独 立 任 务 并 行 执 行 
3. 每 个 执 行 器 都 有 全 新 的  200K 上 下 文 窗 口 （避 免 上 下 文 腐 烂 ）
4. 每 个 任 务 完 成 后 独 立  Git 提 交 
执 行 流 程 ：
Wave 1: Executor A → commit
        Executor B → commit
        ↓
Wave 2: Executor C → commit (依 赖  Wave 1)
        ↓
Verifier: 检 查 代 码 库 是 否 符 合 阶 段 目 标 
---
步 骤  5️⃣：验 证 工 作 
bash
/gsd:verify-work 1
作 用 ： 用 户 验 收 测 试  (UAT)
- 系 统 会 展 示 完 成 的 功 能 
- 你 可 以 手 动 测 试 
- 如 果 发 现 问 题 ，系 统 会 自 动 诊 断 并 创 建 修 复 计 划 
---
步 骤  6️⃣：完 成 里 程 碑 
bash
/gsd:audit-milestone    验 证 里 程 碑 是 否 完 成 
/gsd:complete-milestone 归 档 里 程 碑 ，创 建  release tag
/gsd:new-milestone v2.0 开 始 下 一 个 版 本 周 期 
---

🎓 七 、核 心 设 计 理 念 
1. 上 下 文 工 程 
GSD 通 过 精 心 设 计 的 文 件 管 理 上 下 文 ，避 免  AI"忘 记 "重 要 信 息 ：
| 文 件  | 作 用  | 生 命 周 期  |
|------|------|----------|
| PROJECT.md | 项 目 愿 景 ，始 终 加 载  | 整 个 项 目  |
| REQUIREMENTS.md | 需 求 追 踪 （分  v1/v2/范 围 外 ） | 整 个 项 目  |
| ROADMAP.md | 分 阶 段 路 线 图  | 整 个 项 目  |
| STATE.md | 决 策 、阻 塞 、立 场 记 忆  | 整 个 项 目  |
| CONTEXT.md | 实 现 偏 好 （视 觉 、API 等 ） | 每 阶 段  |
| RESEARCH.md | 技 术 栈 、架 构 、陷 阱 研 究  | 每 阶 段  |
| PLAN.md | XML 结 构 原 子 任 务  | 每 阶 段  |
| SUMMARY.md | 执 行 总 结 和 变 更  | 每 阶 段  |
2. XML 提 示 格 式 化 
每 个 计 划 都 是 结 构 化  XML，例 如 ：
xml
<task type="auto">
  <name>Create login endpoint</name>
  <files>src/app/api/auth/login/route.ts</files>
  <action>Use jose for JWT tokens. Store in HTTP-only cookie.</action>
  <verify>curl -X POST http://localhost:3000/api/auth/login -d '{"email":"test@example.com","password":"test123"}'</verify>
  <done>Valid credentials return Set-Cookie header. Invalid returns 401 with error message.</done>
</task>
这 种 结 构 让  AI 清 晰 理 解 ：
- 做 什 么 （name）
- 改 哪 些 文 件 （files）
- 怎 么 做 （action）
- 如 何 验 证 （verify）
- 完 成 标 准 （done）
3. 多  Agent 编 排 
| Agent | 数 量  | 作 用  |
|-------|------|------|
| 研 究 者  | 4 并 行  | 技 术 栈 、功 能 、架 构 、陷 阱  |
| 规 划 者  | 1 | 创 建 任 务 计 划  |
| 检 查 器  | 1 | 验 证 计 划 是 否 符 合 需 求  |
| 执 行 者  | 多 波 次  | 执 行 任 务 ，每 个  200K 上 下 文  |
| 验 证 器  | 1 | 独 立 检 查 完 成 质 量  |
4. 原 子  Git 提 交 
每 个 任 务 独 立 提 交 ，便 于 ：
- ✅ git bisect 快 速 定 位 问 题 
- ✅ 独 立 回 滚 某 个 功 能 
- ✅ 清 晰 的 提 交 历 史 
- ✅ Code Review 更 精 准

⚠️ 常 见 问 题 
Q: 为 什 么 需 要  --dangerously-skip-permissions？
A: GSD 会 频 繁 执 行  git commit、date、echo 等 命 令 ，每 次 确 认 会 严 重 打 断 工 作 流 。如 果 你 坚 持 要 用 权 限 控 制 ，可 以 在  .claude/settings.json 中 配 置 允 许 的 命 令 列 表 。
Q: 可 以 用 免 费 的 模 型 吗 ？
A: 可 以 ！在  /gsd:settings 中 选 择  budget 模 型 配 置 ，会 使 用  Haiku 等 更 便 宜 的 模 型 。适 合 预 算 有 限 或 简 单 任 务 。
Q: 适 合 小 项 目 吗 ？
A: 适 合 ！你 可 以 只 使 用 核 心 流 程 ，跳 过 研 究 和 讨 论 步 骤 。对 于 非 常 小 的 任 务 ，使 用  /gsd:quick 命 令 即 可 。
Q: 如 何 更 新  GSD？
A: 运 行  /gsd:update 检 查 更 新 ，或 重 新 运 行  npx get-shit-done-cc@latest
Q: 支 持 哪 些  AI 工 具 ？
A: 
- ✅ Claude Code (主 要 支 持 )
- ✅ OpenCode (开 源 ，支 持 免 费 模 型 )
- ✅ Gemini CLI
- ✅ Codex (通 过  skills 方 式 )
Q: 可 以 和 现 有 工 作 流 整 合 吗 ？
A: 可 以 ！GSD 生 成 的 都 是  Markdown 文 件 ，可 以 和 你 的 文 档 系 统 整 合 。Git 分 支 策 略 也 可 配 置 。
---
🎯 九 、适 合 人 群 
- ✅ 独 立 开 发 者 想 要 描 述 需 求 并 让  AI 正 确 构 建 
- ✅ 不 想 玩 "企 业 角 色 扮 演 "（冲 刺 会 议 、故 事 点 、Jira）的 创 作 者 
- ✅ 需 要 可 靠 、可 扩 展 的  AI 编 码 工 作 流 
- ✅ 希 望  AI 产 出 高 质 量 、可 维 护 代 码 的 开 发 者 
不 适 合 ：
- ❌ 想 要 完 全 手 动 控 制 每 个 细 节 的 人 
- ❌ 已 经 有 一 套 成 熟 开 发 流 程 的 大 团 队

本 文 档 是  Claude Code 的  MCP (Model Context Protocol) 官 方 文 档 。MCP 是 一 个 开 源 的  AI-工 具 集 成 标 准 ，允 许  Claude Code 连 接 到 数 百 个 外 部 工 具 、数 据 库 和  API。文 档 详 细 介 绍 了 如 何 安 装 、配 置 和 管 理  MCP 服 务 器 ，以 及 三 种 传 输 方 式 （HTTP、SSE、stdio）和 三 种 配 置 范 围 （本 地 、项 目 、用 户 ）。
内 容 结 构 
1. MCP 简 介  - 什 么 是  MCP 及 其 用 途 
2. 安 装 方 式  - 三 种 传 输 方 式 的 安 装 指 南 
3. 配 置 范 围  - Local、Project、User 三 级 作 用 域 
4. 实 际 示 例  - Sentry、GitHub、PostgreSQL 等 集 成 案 例 
5. 身 份 验 证  - OAuth 2.0 认 证 配 置 
6. 高 级 功 能  - 工 具 搜 索 、资 源 引 用 、命 令 提 示 
7. 企 业 管 理  - managed-mcp.json 和 允 许 /拒 绝 列 表 
关 键 知 识 点 
1. MCP 的 三 种 传 输 方 式 
| 方 式  | 适 用 场 景  | 命 令 示 例  |
|------|----------|----------|
| HTTP | 云 服 务 、远 程  API | claude mcp add --transport http notion https://mcp.notion.com/mcp |
| SSE | 服 务 器 推 送 事 件  | claude mcp add --transport sse asana https://mcp.asana.com/sse |
| stdio | 本 地 进 程 、系 统 访 问  | claude mcp add --transport stdio db -- npx -y @bytebase/dbhub |
2. 三 种 配 置 范 围 
| 范 围  | 存 储 位 置  | 适 用 场 景  |
|------|----------|----------|
| Local | /.claude.json (项 目 路 径 ) | 个 人 开 发 、敏 感 凭 证 、实 验 性 配 置  |
| Project | .mcp.json (项 目 根 目 录 ) | 团 队 共 享 、版 本 控 制 、协 作 工 具  |
| User | /.claude.json | 跨 项 目 个 人 工 具 、常 用 服 务  |
3. 环 境 变 量 扩 展 
.mcp.json 支 持 环 境 变 量 语 法 ：
- ${VAR} - 展 开 变 量 值 
- ${VAR:-default} - 变 量 未 设 置 时 使 用 默 认 值 
示 例 ：
json
{
  "mcpServers": {
    "api-server": {
      "type": "http",
      "url": "${APIBASEURL:-https://api.example.com}/mcp",
      "headers": {
        "Authorization": "Bearer ${APIKEY}"
      }
    }
  }
}
重 要 概 念 
| 术 语  | 解 释  |
|------|------|
| MCP | Model Context Protocol，AI-工 具 集 成 的 开 源 标 准  |
| MCP Server | 提 供  tools/prompts/resources 的 服 务 端 程 序  |
| Transport | MCP 连 接 的 通 信 方 式 ：HTTP/SSE/stdio |
| Scope | MCP 配 置 的 作 用 域 级 别  |
| Tool Search | 当  MCP 工 具 定 义 超 过 上 下 文 窗 口  10% 时 自 动 启 用 ，动 态 加 载 工 具  |
| MCP Resources | 可 通 过  @ 提 及 引 用 的 资 源 ，类 似 文 件 引 用  |
数 据 与 事 实 
- 输 出 警 告 阈 值 : MCP 工 具 输 出 超 过  10,000 tokens 时 显 示 警 告 
- 默 认 输 出 限 制 : 25,000 tokens
- 可 通 过 环 境 变 量 调 整 : MAXMCPOUTPUTTOKENS
- Tool Search 自 动 触 发 : 默 认  10% 上 下 文 阈 值 
- 支 持 的 模 型 : Sonnet 4+、Opus 4+（Haiku 不 支 持 工 具 搜 索 ）

claude mcp get github
删 除 服 务 器 
claude mcp remove github
在  Claude Code 中 检 查 服 务 器 状 态 
/mcp
启 动  Claude 作 为  MCP 服 务 器 
claude mcp serve

=== ID: 2207 | Title: 1. 克隆仓库（放用户配置目录，方便更新） | Path: 人工智能/claude code/Superpowers安装.md | Words: 4504 ===
在  Claude Code 和  OpenCode 中 安 装 与 使 用  Superpowers 的 方 式 有 明 显 区 别 ：Claude Code 走 官 方 插 件 市 场 （最 简 单 、最 推 荐 ），而  OpenCode 需 要 手 动 克 隆 仓 库  + symlink（因 为 它 目 前 没 有 内 置 的 插 件 市 场 支 持 ）。下 面 我 把 两 个 平 台 的 完 整 步 骤 、注 意 事 项 、验 证 方 式 、使 用 方 式 、常 见 问 题 及 边 缘 情 况 都 详 细 列 出 来 （基 于  2026 年  2 月 的 最 新 仓 库 状 态 ）。
1. 在  Claude Code 中 安 装 与 使 用 （推 荐 方 式 ，2 条 命 令 搞 定 ）
前 提   
- 你 已 经 在 用  Claude Code（命 令 行 版 ，Anthropic 官 方 工 具 ）。
- 确 保  Claude Code 版 本  ≥ 2.0.13（插 件 系 统 支 持 较 完 整 ）。
- 网 络 正 常 （安 装 时 会 从  GitHub 拉 取 ）。
安 装 步 骤 （在  Claude Code 终 端 里 直 接 输 入 ）：
1. 先 注 册  Superpowers 的 专 用  marketplace（只 需 做 一 次 ）：
   
   /plugin marketplace add obra/superpowers-marketplace
   
2. 然 后 从 这 个  marketplace 安 装  superpowers 插 件 ：
   
   /plugin install superpowers@superpowers-marketplace
   
更 新 方 式 （以 后 想 升 级 时 用 ）：
/plugin update superpowers
（它 会 自 动 拉 取 最 新 技 能 和 修 复 ）
验 证 安 装 成 功 ：
- 退 出 当 前  Claude Code 会 话 （或 新 开 一 个 终 端 ）。
- 重 新 启 动  Claude Code，进 入 项 目 目 录 。
- 输 入 一 句 能 触 发  Superpowers 的 自 然 语 言 指 令 ，例 如 ：
  - “帮 我 规 划 一 个 用 户 登 录 功 能 ，包 括 前 后 端 ”
  - “help me plan this feature: add OAuth2 login”
  - “let's debug why this test is failing”
- 如 果 看 到 它 自 动 开 始  brainstorming → 输 出 结 构 化 问 题  → 等 待 你 确 认  → 生 成  Implementation Plan，说 明 成 功 了 。
日 常 使 用 方 式 （几 乎 零 手 动 操 作 ）：
- 不 需 要 额 外 命 令 激 活 ，Superpowers 是 “被 动 触 发  + 强 制 执 行 ”的 。
- 你 只 要 正 常 用 自 然 语 言 描 述 需 求 ，它 就 会 根 据 上 下 文 自 动 调 用 合 适 的  skill（比 如 需 求 不 清 晰  → brainstorming；要 写 代 码  → 先  plan → TDD → subagents → review）。
- 典 型 触 发 词 （可 以 直 接 说 ）：
  - plan / planning / 规 划  / 设 计 
  - implement / 实 现  / 写 代 码 
  - debug / 调 试  / 为 什 么 报 错 
  - review / code review / 检 查 代 码 
  - refactor / 重 构 
- 它 会 强 制 走 完 整 流 程 （你 不 确 认  plan 它 就 不 往 下 写 代 码 ，没 测 试 它 就 不 提 交 ）。
Claude Code 特 有 优 势 ：
- 安 装 最 简 单 ，官 方 支 持 最 好 。
- 更 新 最 方 便 。
- 与  Anthropic 生 态 集 成 最 紧 密 （子 代 理 、hooks 等 都 原 生 兼 容 ）。
2. 在  OpenCode 中 安 装 与 使 用 （手 动  symlink 方 式 ）
前 提   
- 已 安 装  OpenCode（https://opencode.ai 的  CLI 工 具 ，开 源 版  Claude Code 风 格 ）。
- Git 已 安 装 。
- 系 统 为  macOS / Linux（Windows 可 能 需 调 整 路 径 ）。
- 建 议 搭 配  oh-my-opencode 插 件 使 用 （增 强  subagent 和 后 台 任 务 能 力 ，但 非 必 须 ）。
安 装 步 骤 （最 简 单 的 一 步 式 指 令 方 式 ）：
1. 直 接 在  OpenCode 终 端 里 输 入 下 面 这 句 话 ，它 会 自 己 帮 你 执 行 安 装 ：
   
   Fetch and follow instructions from https://raw.githubusercontent.com/obra/superpowers/refs/heads/main/.opencode/INSTALL.md
   
   - OpenCode 会 自 动 读 取 那 个  INSTALL.md，然 后 提 示 或 直 接 帮 你 克 隆 和  symlink。
等 价 的 手 动 完 整 步 骤 （如 果 你 想 自 己 控 制 ）：
bash
1. 克 隆 仓 库 （放 用 户 配 置 目 录 ，方 便 更 新 ）
git clone https://github.com/obra/superpowers.git /.config/opencode/superpowers
2. 创 建  plugins 目 录 （如 果 不 存 在 ）
mkdir -p /.config/opencode/plugins
3. 创 建  symlink 让  OpenCode 发 现 插 件 
rm -f /.config/opencode/plugins/superpowers.js
ln -s /.config/opencode/superpowers/.opencode/plugins/superpowers.js /.config/opencode/plugins/superpowers.js
4. （可 选 但 推 荐 ）symlink 技 能 目 录 ，让 技 能 更 容 易 被 发 现 /更 新 
ln -s /.config/opencode/superpowers/skills /.config/opencode/skills/superpowers
5. 重 启  OpenCode
退 出 当 前  opencode 进 程 ，然 后 重 新 运 行  opencode
更 新 方 式 ：
- 进 入  /.config/opencode/superpowers 目 录 
- git pull
- 如 果 有  symlink 变 化 ，按 上 面 步 骤 重 新  symlink
- 重 启  OpenCode
验 证 安 装 成 功 ：
- 重 启  OpenCode 后 ，进 入 项 目 。
- 输 入 类 似  Claude Code 的 触 发 句 ：
  - “plan a new REST API endpoint for user profile”
  - “debug this failing unit test”
- 应 该 看 到 它 自 动 进 入  Superpowers 流 程 （问 问 题  → plan → TDD 等 ）。
OpenCode 特 有 注 意 事 项  & 边 缘 情 况 ：
- OpenCode 的 技 能 系 统 是 “注 入 式 ”的 ，Superpowers 会 通 过  bootstrap 做 一 些  mapping（比 如  Claude 的 某 些 指 令 在  OpenCode 里 会 映 射 成  updateplan 等 ）。
- 如 果 你 同 时 用 了  oh-my-opencode，Superpowers 和 它 可 以 共 存 ，但 偶 尔 会 出 现  skill 名 称 冲 突 或 工 具 调 用 混 乱 （比 如  findskills 工 具 重 复 ），建 议 优 先 用  Superpowers 的 子 代 理 机 制 。
- OpenCode 没 有  marketplace，所 以 更 新 全 靠  git pull，手 动 性 更 强 。
- 如 果 你 经 常 切 换 项 目 ，建 议 把  superpowers 克 隆 到 全 局 路 径 ，而 不 是 每 个 项 目 单 独  clone。
- 极 少 数 情 况 下  symlink 失 效 （权 限 问 题 或 路 径 变 了 ），可 以 删 掉 所 有  symlink 后 重 新 跑 上 面 命 令 。
两 个 平 台 共 性  & 建 议 
- 不 需 要 手 动 启 用 任 何  skill：安 装 后 它 们 自 动 工 作 ，靠 上 下 文 触 发 。
- 最 有 效 的 启 动 方 式 ：新 开 一 个 干 净 会 话  + 直 接 说 “我 要 实 现 /规 划 /调 试  xxx”，让 它 从 头 走  Superpowers 流 程 。
- 想 临 时 禁 用 ：目 前 没 有 全 局 开 关 ，最 简 单 是 新 开 一 个 不 带  Superpowers 的 会 话 ，或 者 在  prompt 里 明 确 说 “不 要 用  superpowers 流 程 ，直 接 写 代 码 ”。
- 想 自 己 改  skill：所 有  skill 都 在  skills/ 目 录 ，都 是 纯  Markdown + prompt 文 件 ，你 可 以  fork 仓 库 修 改 后 本 地  symlink。
- 常 见 报 错 处 理 ：
  - “plugin not found” → 检 查 命 令 拼 写  / marketplace 是 否 添 加 
  - “skill not triggering” → 换 个 更 明 确 的 触 发 词 （plan/debug/implement）
  - OpenCode symlink 失 效  → 检 查 路 径 是 否 正 确 ，权 限 是 否  755

Claude Code 的 插 件 默 认 安 装 在 用 户 目 录 下 的 配 置 文 件 夹 中 ，具 体 路 径 因 操 作 系 统 而 异 ：

说 明 ：
- 插 件 会 以  superpowers 或 类 似 名 称 的 文 件 夹 形 式 存 放 在 上 述 路 径 中 ；
    
- 若 需 手 动 查 看 ，可 在  Claude Code 中 通 过  /plugin list 命 令 查 看 已 安 装 插 件 的 具 体 路 径 。

OpenClaw、OpenCode 和  Claude Code 都 是 2025–2026年 间 爆 火 的 AI agent类 工 具 ，但 它 们 的 核 心 定 位 、使 用 场 景 和 能 力 边 界 差 异 非 常 大 。
简 单 一 句 话 总 结 三 者 的 本 质 区 别 
- OpenClaw：通 用 型 "能 干 活 的 个 人 /家 庭 /团 队 助 理 "（真 正 执 行 现 实 世 界 操 作 ）

- Claude Code：Anthropic官 方 出 品 的 "最 强 Claude编 程 体 验 "（封 闭 但 极 致 打 磨 ）

| 主 要 用 途  | 生 活 /工 作 自 动 化 （收 邮 件 、日 历 、发 消 息 、订 票 、浏 览 器 操 作 、shell命 令 、本 地 文 件 读 写 等 ） | 纯 编 程 任 务 （写 代 码 、重 构 、debug、写 测 试 、理 解 代 码 库 、多 文 件 修 改 ） | 纯 编 程 任 务 （官 方 定 位 就 是 "Claude写 代 码 "） |

| 自 主 性 /Agent程 度  | 极 高 （可 以 设 定 目 标 后 自 己 循 环 执 行 ） | 高 （支 持 sub-agents、多 轮 自 主 尝 试 修 复 ） | 高 （有 sub-agents、思 考 链 很 强 ） |
| 安 全 性 争 议  | ★☆☆☆☆（技 能 过 于 强 大 ，社 区 出 现 恶 意 skill、越 权 执 行 、被 用 作 恶 意 软 件 载 体 的 案 例 极 多 ） | ★★★★☆（默 认 比 较 保 守 ，可 配 置 ） | ★★★★☆（Anthropic有 较 多 安 全 守 栏 ） |
| 部 署 难 度  | 中 等 （docker一 键 多 ，但 配 置 技 能 /权 限 很 繁 琐 ） | 低 （brew install / curl安 装 ，几 分 钟 跑 起 来 ） | 极 低 （pip install + 登 录 即 可 ） |
| 成 本  | 自 托 管 免 费  + 你 用 的 LLM费 用  | 工 具 免 费  + 你 用 的 LLM费 用  | $20–200/月 订 阅  或  API按 量 付 费  |
| 典 型 用 户  | 想 让 AI"替 我 活 着 的 "人 （高 净 值 、忙 碌 高 管 、极 客 ） | 开 发 者 、想 逃 离 厂 商 锁 定 的 极 客 团 队  | 深 度 Claude用 户 、重 度 依 赖 Anthropic生 态 的 开 发 者  |
| 2026年 社 区 热 度  | 病 毒 式 传 播 ，但 安 全 担 忧 拖 后 腿  | 持 续 上 升 （尤 其 是 厌 倦 Claude锁 定 的 群 体 ） | 仍 然 最 稳 的 编 程 agent标 杆  |
---
核 心 功 能 差 异 总 结 （最 直 观 的 三 个 角 度 ）
1. 你 想 让 AI帮 你 "干 活 "还 是 "写 代 码 "？
- 收 邮 件 、订 机 票 、管 日 历 、自 动 回 复 消 息 、监 控 GitHub issue并 自 动 回 复  → 选 OpenClaw
- 重 构 遗 留 系 统 、跨 文 件 debug、生 成 完 整 功 能 模 块 、跑 测 试 循 环 修 复  → 选 OpenCode 或  Claude Code
2. 你 是 否 已 经 被 Anthropic深 度 绑 定 ？
- 已 经 买 了 Claude Pro/Max，只 想 最 好 的 Claude编 程 体 验 ，不 想 折 腾  → Claude Code
- 讨 厌 厂 商 锁 定 ，想 用 Claude时 用 Claude，想 省 钱 用 本 地 /DeepSeek/Gemini时 切 过 去  → OpenCode
3. 你 对 "AI拿 枪 指 着 自 己 电 脑 "有 多 大 接 受 度 ？
- 愿 意 承 担 极 大 安 全 风 险 换 取 最 真 实 的 "AI替 我 活 " → OpenClaw（但 强 烈 建 议 沙 箱 /小 号 /监 控 ）
- 想 要 强 大 但 可 控 的 执 行 能 力  → OpenCode（编 程 域 最 强 开 源 选 项 ）
---
2026年 2月 真 实 使 用 建 议 排 序 （按 场 景 ）
- 只 想 写 代 码 ，且 预 算 充 足 、追 求 极 致 体 验  → Claude Code > OpenCode（用 Claude模 型 ）
- 写 代 码 ，但 想 省 钱 /多 模 型 /本 地 跑 /自 托 管  → OpenCode >> Claude Code
- 想 要 真 正 的 "生 活 /工 作 助 理 "而 非 编 程 工 具  → OpenClaw（但 请 务 必 阅 读 安 全 警 告 ，社 区 恶 意 skill泛 滥 ）
- 两 者 都 想 要  → 大 多 数 人 最 终 会 同 时 跑 ：OpenClaw管 生 活  + OpenCode/Claude Code管 编 程 
---
一 句 话 结 论 
- OpenClaw 是 "AI秘 书 /管 家 "
- OpenCode 是 "开 源 版 Claude Code + 模 型 自 由 "
- Claude Code 是 "官 方 最 强 Claude编 程 副 驾 "
看 你 到 底 是 想 让 AI帮 你 活 得 更 轻 松 ，还 是 代 码 写 得 更 快 更 好 ，答 案 就 很 清 晰 了 。

Claude Code 使 用 技 巧 

核 心 理 念 

Claude Code 是  Anthropic 推 出 的 命 令 行  AI 编 程 代 理 ，它 直 接 在 终 端 中 运 行 ，能 够 理 解 整 个 代 码 库 并 执 行 复 杂 的 编 程 任 务 。掌 握 使 用 技 巧 可 以 大 幅 提 升 开 发 效 率 。

---

一 、CLAUDE.md — 项 目 记 忆 文 件 

1. 作 用 
CLAUDE.md 是  Claude Code 的 "项 目 记 忆 "，类 似 于 给  AI 的  system prompt。每 次 对 话 开 始 时 ，Claude Code 会 自 动 读 取 该 文 件 ，从 而 了 解 项 目 的 背 景 、规 范 和 偏 好 。

2. 三 个 层 级 

| 文 件 位 置  | 作 用 范 围  | 适 用 内 容  |
|---------|---------|---------|
| `~/.claude/CLAUDE.md` | 全 局  | 个 人 偏 好 、通 用 编 码 规 范  |
| `项 目 根 目 录 /CLAUDE.md` | 当 前 项 目  | 项 目 架 构 、技 术 栈 、编 码 约 定  |
| `子 目 录 /CLAUDE.md` | 特 定 模 块  | 模 块 级 规 范 、注 意 事 项  |

3. CLAUDE.md 应 该 写 什 么 

```markdown
CLAUDE.md 示 例 

项 目 概 述 
这 是 一 个  Next.js 15 + TypeScript 的 电 商 后 台 管 理 系 统 。

技 术 栈 
- 框 架 : Next.js 15 (App Router)
- 语 言 : TypeScript (strict mode)
- 数 据 库 : PostgreSQL + Prisma ORM
- 样 式 : Tailwind CSS
- 测 试 : Vitest + Testing Library

编 码 规 范 
- 使 用 函 数 式 组 件 ，禁 止  class 组 件 
- 所 有 组 件 必 须 有  TypeScript 类 型 定 义 
- 文 件 命 名 : 组 件 用  PascalCase，工 具 函 数 用  camelCase
- 每 个 文 件 不 超 过  200 行 

常 用 命 令 
- 开 发 : `npm run dev`
- 构 建 : `npm run build`
- 测 试 : `npm test`
- Lint: `npm run lint`
- 数 据 库 迁 移 : `npx prisma migrate dev`

架 构 说 明 
- `src/app/` — 页 面 路 由 
- `src/components/` — 共 享 组 件 
- `src/lib/` — 工 具 函 数 和 业 务 逻 辑 
- `src/hooks/` — 自 定 义  Hooks
- `prisma/` — 数 据 库  schema

注 意 事 项 
- 不 要 修 改  prisma/schema.prisma，需 要 先 讨 论 
- API 路 由 必 须 做 输 入 验 证  (zod)
- 所 有 数 据 库 操 作 通 过  Prisma Client，禁 止 原 生  SQL
```

4. 技 巧 
- 在  CLAUDE.md 中 列 出 **常 用 命 令 **，这 样 你 可 以 直 接 说 "运 行 测 试 "而 不 用 每 次 指 定 命 令 
- 写 明 **不 要 做 什 么 **（禁 止 事 项 ）和 **应 该 做 什 么 **同 样 重 要 
- 定 期 更 新 ，保 持 与 项 目 实 际 状 态 一 致 

---

二 、高 效 对 话 技 巧 

1. 明 确 具 体 ，一 次 说 清 
```
❌ 帮 我 写 个 组 件 
✅ 在  src/components/UserCard.tsx 创 建 一 个 用 户 卡 片 组 件 ，接 收  name, email, avatar 三 个  props，用  Tailwind CSS 样 式 ，支 持 点 击 跳 转 到 用 户 详 情 页 
```

2. 分 步 骤 拆 解 复 杂 任 务 
```
1. 先 帮 我 设 计 数 据 库  schema
2. 然 后 创 建  API 路 由 
3. 最 后 写 前 端 页 面 调 用  API
```

3. 善 用 上 下 文 引 用 
```
参 考  src/components/Header.tsx 的 样 式 风 格 ，创 建 一 个  Footer 组 件 
```

4. 让  Claude Code 先 理 解 再 行 动 
```
先 看 看 这 个 项 目 的 结 构 ，告 诉 我 你 的 理 解 ，然 后 再 帮 我 添 加 新 功 能 
```

|------|------|

| `/clear` | 清 除 当 前 对 话 上 下 文  |

| `/cost` | 查 看 当 前 会 话 的  token 消 耗 和 费 用  |

| `/login` | 切 换 账 号  |
| `/status` | 查 看 当 前 状 态  |

| 快 捷 键  | 说 明  |

3. 权 限 模 式 

```bash
允 许 所 有 工 具 调 用 （适 合 信 任 的 环 境 ）
claude --dangerously-skip-permissions

指 定 工 作 目 录 
claude --cwd /path/to/project

继 续 上 次 的 对 话 
claude --continue
claude --resume

传 入 初 始 提 示 
claude -p "解 释 这 个 项 目 的 架 构 "
```

四 、高 级 使 用 技 巧 

1. 子 代 理 （Sub-Agents）
Claude Code 可 以 启 动 子 代 理 并 行 执 行 独 立 任 务 ，例 如 同 时 搜 索 多 个 文 件 或 并 行 进 行 代 码 审 查 。

2. Git 集 成 
```
创 建 一 个 新 分 支  feature/user-auth，然 后 在 这 个 分 支 上 实 现 用 户 认 证 功 能 
```

Claude Code 会 自 动 ：
- 创 建 分 支 
- 编 写 代 码 
- 运 行 测 试 
- 提 交 代 码 

3. 多 文 件 编 辑 
```
重 构 整 个  src/utils/ 目 录 ，将 所 有 工 具 函 数 从  JavaScript 转 换 为  TypeScript，添 加 类 型 定 义 
```

4. 代 码 审 查 
```
审 查 最 近 一 次  git commit 的 所 有 改 动 ，检 查 潜 在 的  bug 和 性 能 问 题 
```

5. 调 试 助 手 
```
这 个 测 试 报 错 了  [粘 贴 错 误 信 息 ]，帮 我 找 到 问 题 所 在 并 修 复 
```

6. 文 档 生 成 
```
为  src/api/ 目 录 下 所 有  API 端 点 生 成  OpenAPI 文 档 
```

五 、配 置 优 化 

1. 全 局 配 置 文 件 

位 置 ：`~/.claude/settings.json`

```json
{
  "permissions": {
    "allow": [
      "Bash(npm run *)",
      "Bash(git *)",
      "Read",
      "Write"
    ],
    "deny": [
      "Bash(rm -rf *)"
    ]
  }
}
```

2. 项 目 级 配 置 

位 置 ：`项 目 根 目 录 /.claude/settings.json`（可 提 交 到  git）

```json
{
  "permissions": {
    "allow": [
      "Bash(npm test *)",
      "Bash(npx prisma *)"
    ]
  }
}
```

3. 自 定 义 斜 杠 命 令 

在  `.claude/commands/` 目 录 下 创 建  `.md` 文 件 ，即 可 定 义 可 复 用 的 自 定 义 命 令 ：

```
.claude/
  commands/
    review.md       → /project:review
    test.md         → /project:test
    deploy.md       → /project:deploy
```

命 令 文 件 内 容 示 例 （`.claude/commands/review.md`）：
```markdown
请 对 当 前  git diff 中 的 所 有 改 动 进 行 代 码 审 查 ：
1. 检 查 是 否 有  bug
2. 检 查 是 否 符 合 项 目 编 码 规 范 
3. 检 查 是 否 有 性 能 问 题 
4. 检 查 是 否 有 安 全 漏 洞 
5. 给 出 改 进 建 议 
```

使 用 时 输 入  `/project:review` 即 可 。

---

六 、最 佳 实 践 

1. 迭 代 式 开 发 
```
第 1步 ：帮 我 搭 建 项 目 骨 架 
第 2步 ：实 现 用 户 注 册 登 录 
第 3步 ：添 加 权 限 管 理 
...
```
每 次 做 完 一 步 ，确 认 没 问 题 再 继 续 下 一 步 。

2. 让  Claude 解 释 再 执 行 
```
先 告 诉 我 你 打 算 怎 么 实 现 这 个 功 能 ，我 确 认 后 再 动 手 
```

3. 善 用  /compact 压 缩 上 下 文 
对 话 太 长 时 ，使 用  `/compact` 压 缩 历 史 ，保 持 上 下 文 窗 口 高 效 利 用 。

4. 明 确 指 定 文 件 路 径 
```
在  src/components/Header.tsx 的 第  15 行 之 后 添 加 一 个 导 航 菜 单 
```

5. 测 试 驱 动 
```
先 为 用 户 注 册 功 能 写 测 试 用 例 ，然 后 实 现 功 能 让 测 试 通 过 
```

6. 保 持  CLAUDE.md 更 新 
项 目 架 构 或 规 范 变 更 时 ，及 时 更 新  CLAUDE.md，让  Claude Code 始 终 了 解 最 新 情 况 。

7. 安 全 意 识 
- 不 要 在  CLAUDE.md 中 存 放 密 钥 或 敏 感 信 息 
- 使 用 权 限 配 置 限 制 危 险 操 作 
- 在 生 产 环 境 相 关 操 作 前 做 好 备 份

七 、常 见 场 景 示 例

请 帮 我 了 解 这 个 项 目 的 整 体 架 构 ，包 括 ：

[粘 贴 错 误 信 息 ]
```

```
帮 我 分 析  src/components/Dashboard.tsx 的 性 能 问 题 ，特 别 是 ：

2. 大 列 表 的 虚 拟 滚 动

```
为 项 目 添 加  i18n 支 持 ：

```
