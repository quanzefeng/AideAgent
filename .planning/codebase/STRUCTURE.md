## Directory Purposes

**`desktop/` (root):**
- Purpose: Electron app home. `main.mjs` is the entry; `preload.cjs` is the bridge; everything else is either a "store" (root-level stateful module) or a peer directory.
- Contains: `main.mjs`, `preload.cjs`, `package.json`, `tsconfig.json`, `eslint.config.js`, `vitest.config.mjs`, `playwright.config.mjs`, root-level store modules.

**`desktop/core/`:**
- Purpose: Main-process agent runtime. Every file here is `import`ed by either `main.mjs` or `ipc-handlers.mjs` (or each other). Modules are side-effect-free ES modules; state lives in `state.mjs`.
- Contains: 15 `.mjs` files (one is `wechat-bridge.mjs.bak` backup).

**`desktop/renderer/`:**
- Purpose: Renderer process UI. Vanilla JS + global CSS, no bundler.
- Contains: `index.html`, `style.css`, `app.js`, `translations.js`, `global.d.ts`, `avatar.jpg`, `modules/`.

**`desktop/renderer/modules/`:**
- Purpose: One file per UI concern. Imported either as side-effect (`import './modules/font-settings.mjs'`) or as named factories (`initKnowledgeBase`, `initMemoryPanel`, etc.).
- Contains: 13 `.mjs` files.

**`desktop/search-engine/`:**
- Purpose: Free meta-search implementation used by the `web_search` tool when no Tavily key is configured.
- Contains: `index.mjs` (entry), `debug.mjs`, `inspect.mjs`, `inspect2.mjs`, `test.mjs` (utility/debug scripts).

**`desktop/models/`:**
- Purpose: Bundled ML model files for knowledge-base embeddings.
- Contains: `all-MiniLM-L6-v2/` (ONNX embedding model used by `knowledge-store.mjs`).

**`desktop/scripts/`:**
- Purpose: Build / postinstall scripts.
- Contains: `download-model.mjs` (referenced by `package.json:postinstall`).

**`desktop/test/`:**
- Purpose: Unit tests (Vitest) + end-to-end tests (Playwright).
- Contains: 12 vitest specs (`*.test.mjs`) + `e2e-full.test.mjs` + `e2e/` subdirectory.

**`desktop/test/e2e/`:**
- Purpose: Playwright e2e specs.
- Contains: `smoke.test.mjs`, `agent-name.test.mjs`, `kb.test.mjs`, `memory.test.mjs`, `skills.test.mjs`, `font-lang.test.mjs`, `prompt.test.mjs`.

**`desktop/dist/`:**
- Purpose: Pre-built CLI artifact copied into releases via `package.json:build.extraResources`. Generated.

**`desktop/release/`:**
- Purpose: electron-builder output. Generated.

**`desktop/playwright-report/`, `desktop/test-results/`, `desktop/.playwright-mcp/`, `desktop/.vs/`:**
- Purpose: Tool-generated caches; not source.

## Naming Conventions

**Files:**
- `.mjs` for main-process modules, ES modules with `"type": "module"` in `package.json`. The codebase uses `.mjs` everywhere **except** the preload script.
- `.cjs` for `preload.cjs` only (CommonJS, required by Electron's preload context in this Electron version).
- `.bak` suffix for backup files (`wechat-bridge.mjs.bak`); not in normal use.
- `.d.ts` for ambient type declarations (`renderer/global.d.ts`).
- `kebab-case` for everything except a few top-level filenames which use single-word lowercase (`session-db.mjs`, `memory-store.mjs`).

**Directories:**
- `kebab-case` (`core/`, `renderer/`, `search-engine/`, `playwright-report/`, `test-results/`).

**Modules inside `core/` and `renderer/modules/`:**
- `kebab-case.mjs` filenames (`agent-loop.mjs`, `system-prompt.mjs`, `memory-panel.mjs`, etc.).

**JavaScript identifiers:**
- Functions: `camelCase` (`agentLoop`, `runTool`, `selectRelevantMemories`, `buildSystemPrompt`).
- Types: `PascalCase` (`ApiConfig`, `Message`).
- Constants: `SCREAMING_SNAKE_CASE` (`MAX_TURNS`, `MAX_CONTINUATIONS`, `CONTEXT_WINDOW`, `CONTEXT_COMPRESS_PCT`, `TOKEN_BUDGET_WARN`, `TOKEN_BUDGET_HARD`, `PLAN_MODE_READONLY`, `SUB_AGENT_TOOL_NAMES`, `SUB_AGENT_MAX_TURNS`, `WX_BASE`, `WX_BOT_TYPE`, `WX_POLL_TIMEOUT`).
- Maps / Sets / singletons: `_underscoreCamelCase` for module-private and `lowerCamelCase` for exported (`pendingPerms`, `_subAgentCtrls`, `_surfacedMemories`, `_askResolvers`, `taskStore`, `_todoList`).
- Private (file-scoped) helpers: `_leadingUnderscore` (`_sysPromptCache`, `_contextBlockBaseCache`, `_cachedToolDefs`, `_cachedToolKey`, `_initPromptStorePath`, `_kbPanelLoaded`, `_skillsPanelLoaded`, `_permResolve`, `_thinkBuffer`, `_afterToolCall`).

## Where to Add New Code

**New main-process module (e.g. a new agent subsystem):**
- Implementation: `desktop/core/<name>.mjs` (kebab-case).
- If it owns new mutable cross-module state, put the state in `desktop/core/state.mjs` with paired `getX`/`setX` exports.
- Register any new IPC channels inside `desktop/core/ipc-handlers.mjs:45` `registerIpcHandlers()`.

**New IPC channel:**
- Implementation: add handler inside `desktop/core/ipc-handlers.mjs:45` `registerIpcHandlers()`.
- Renderer bridge: add a paired `invoke` / `on` wrapper in `desktop/preload.cjs:3` `contextBridge.exposeInMainWorld("aideagent", {...})`.
- Renderer consumer: call `window.aideagent.<name>(...)` from `app.js` or a sub-module.

**New builtin tool (LLM-callable):**
- Tool schema: add an entry to `TOOL_DEFS` in `desktop/core/tool-definitions.mjs`.
- Dispatch: add a `case "<name>":` to the switch in `desktop/core/tool-executor.mjs:181` `runTool(tc)`.
- If it must be plan-mode-restricted: add the name to `PLAN_MODE_READONLY` in `desktop/core/state.mjs:154` (or leave it out to keep it writable).
- If sub-agents must not call it: add to `SUB_AGENT_TOOL_NAMES` in `desktop/core/state.mjs:166` (presence = allowed).
- System prompt mention: append to `DEFAULT_PROMPT` in `desktop/core/system-prompt.mjs:25`.

**New renderer panel (settings tab):**
- Implementation: `desktop/renderer/modules/<name>.mjs` exposing an `init...` factory.
- DOM: append markup to `desktop/renderer/index.html`.
- Wiring: import the factory at the top of `app.js` and call it once at startup.

**New sub-agent entry:**
- Sub-agent already exists (`Agent` tool). Add a new LLM-callable tool if you need a new capability, or call `runSubAgent(description, prompt, subAgentId)` from `desktop/core/tool-executor.mjs:473`.

**New utility / shared helper:**
- Main-process shared helper: place in a new `desktop/core/<name>.mjs` and export directly.
- Renderer shared helper: `desktop/renderer/modules/helpers.mjs`.

**New store (persistent state):**
- Implementation: `desktop/<name>-store.mjs` (matches the existing pattern: `session-db.mjs`, `memory-store.mjs`, `skills-store.mjs`, `knowledge-store.mjs`).
- IPC: add handlers to `desktop/core/ipc-handlers.mjs`.
- Preload: add wrappers to `desktop/preload.cjs`.

**New external integration (e.g. new search provider, new chat provider):**
- Format adapter (if LLM): add a new `xxxCall` function in `desktop/core/format-adapters.mjs`.
- Add a new case to `agentLoop`'s `apiFormat === "..."` selector at `desktop/core/agent-loop.mjs:352`.
